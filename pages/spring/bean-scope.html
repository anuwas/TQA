Bean Scope

There are two basic bean scope is available.

  1. Singleton - Only once per spring container.
  2. Prototype - New bean create with every request or reference.

 

Singleton is the default in spring. So during the initialization of Application context, beans are also initialize, it doesn't wait for getBean. But if it is the prototype then spring waits for the getBean to initialize the beans.

 

<bean id="triangle" class="com.anu.tqa.autowiring.Triangle" scope="singleton"></bean>

<bean id="triangle" class="com.anu.tqa.autowiring.Triangle" scope="prototype"></bean>

 

This singleton is not exactly like the singleton pattern, cause this singleton is dependent on node , so per node 1 object , and it can be multiple node in single jvm.

 

 

 

There is another bean scope available that is Web-aware Context Bean Scope. This is using for web application.

Web-aware Context Bean Scope

  1. Request : New bean per servlet request
  2. Session : New bean per session
  3. Global Session : New bean per global HTTP Session (portlet context)
