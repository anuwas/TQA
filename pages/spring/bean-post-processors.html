  1. Bean Post Processors

BeanPostProcess are class that tells spring , that there are some processing spring need to do after initializing the bean, this is why its named as BeanPostProcessors. What happen if we write some code in BeanPostProcessors , spring execute those code after bean executes those code after initializing each and every bean. So this is something that we can use to extend some functionality.

So a couple of significant point of BeanPostProcessor . First of all the method a single method runs after every initialization of beans of the spring xml. BeanPostProcessor is a separate class.

This is very helpful for configuration related things.

18.1 DisplayNameBeanPostProcessor.java

public class DisplayNameBeanPostProcessor implements BeanPostProcessor{

 

       //this method called after initializing

       //first argument is bean itself and second argument is string

       @Override

       public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {

              System.out.println("After bean Initialization , Bean "+beanName);

              return bean;

       }

 

       //This method called before initializing

       //first argument is bean itself and second argument is string

       @Override

       public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {

              System.out.println("Before bean Initialization , Bean "+beanName);

              return bean;

       }

}

  1. Bean Factory Post Processors

One of the extensions of spring is bean factory post processors. We can write our own bean factory post processor. Bean factory provides the entire bean. To implement bean factory post processor need to write a separate class.

19.1 MyBeanFactoryPostProcessor.java

 

public class MyBeanFacoryPostProcessor implements BeanFactoryPostProcessor{

 

       @Override

       public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFacotry) throws BeansException {

              System.out.println("BeanFacoryPost Processor Executing ");

       }

}

We can override the default functionality of bean factory post processor. One of the example of BeanFactoryPostProcessor is PropertyPlaceHolderConfigurer . Like in beans.xml file different values are abliable. No I want to read those values from a property file.

 19.3 PropertyPlaceHolderConfigurer

One of the examples of BeanFactoryPostProcess is PropertyPlaceHolderConfigurer. Now we are going to put some placeholder in bean.xml file instead of value. And we will put those value in a property file. And PropertyPlaceHolderConfigurer will read those values from property file and put those values in the placeholder which is declared in bean.xml file.

19.3.1 pointconfig.properties

pointA.pointX=0

pointA.pointY=0

** Please the pointconfig.properties file in src folder.
