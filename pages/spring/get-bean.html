Bean Understanding (getBean)

When we initialize application context, then it reads the bean XML and looks that are the beans available in the bean XML and actually creates by itself. Note that there is still no call to getBean(). Without any call the Application context during initialize itself initializes the bean which is configured in the XML. Now after that when the getBean() called then the Application context handed over the the bean that which it had created earlier.

So the by default the creation of the object initialize when Application context get initialize, Not when the getBean() calling. This is the default behaviour of the spring.

This can be configured in different way, It can be configure like when getBean() called then initialize happen. We can also configure the the Application context initialize when the getBean() happen.

But if we didn't do anything the by default the Application context initialize the beans that are available in XML during the initialization of Application context. And when the getBean() happen then Application context hand over the bean that is already created.

This leads to the concepts of Scope.
