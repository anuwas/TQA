  1.  Bean Understanding (getBean)

When we initialize application context, then it reads the bean XML and looks that are the beans available in the bean XML and actually creates by itself. Note that there is still no call to getBean(). Without any call the Application context during initialize itself initializes the bean which is configured in the XML. Now after that when the getBean() called then the Application context handed over the the bean that which it had created earlier.

So the by default the creation of the object initialize when Application context get initialize, Not when the getBean() calling. This is the default behaviour of the spring.

This can be configured in different way, It can be configure like when getBean() called then initialize happen. We can also configure the the Application context initialize when the getBean() happen.

But if we didn't do anything the by default the Application context initialize the beans that are available in XML during the initialization of Application context. And when the getBean() happen then Application context hand over the bean that is already created.

This leads to the concepts of Scope.

 

  1.  Bean Scope

There are two basic bean scope is available.

  1. Singleton - Only once per spring container.
  2. Prototype - New bean create with every request or reference.

 

Singleton is the default in spring. So during the initialization of Application context, beans are also initialize, it doesn't wait for getBean. But if it is the prototype then spring waits for the getBean to initialize the beans.

 

<bean id="triangle" class="com.anu.tqa.autowiring.Triangle" scope="singleton"></bean>

<bean id="triangle" class="com.anu.tqa.autowiring.Triangle" scope="prototype"></bean>

 

This singleton is not exactly like the singleton pattern, cause this singleton is dependent on node , so per node 1 object , and it can be multiple node in single jvm.

 

 

 

There is another bean scope available that is Web-aware Context Bean Scope. This is using for web application.

Web-aware Context Bean Scope

  1. Request : New bean per servlet request
  2. Session : New bean per session
  3. Global Session : New bean per global HTTP Session (portlet context)

 

  1.  ApplicationContextAware

To get access ApplicationContext in some class needs to implement ApplicationContextAware interface in that class.

By doing this any class can access the ApplicationContext and use the different properties provided by the Application context.

15.1 Triangle.java

public class Triangle implements ApplicationContextAware,BeanNameAware{

      

       private Point pointA;

       private Point pointB;

       private Point pointC;

       private ApplicationContext context = null;

      

       public Point getPointA() {

              return pointA;

       }

 

       public void setPointA(Point pointA) {

              this.pointA = pointA;

       }

 

       public Point getPointB() {

              return pointB;

       }

      

       public void setPointB(Point pointB) {

              this.pointB = pointB;

       }

 

       public Point getPointC() {

              return pointC;

       }

      

       public void setPointC(Point pointC) {

              this.pointC = pointC;

       }

 

       public void draw() {

              System.out.println("Triangle Draw ,"

                           + " Point A "+getPointA().getPointX()+","+getPointA().getPointY()

                           + ", Pint B "+getPointB().getPointX()+", "+getPointB().getPointY()

                           + ", Point C "+getPointC().getPointX()+", "+ getPointC().getPointY());

       }

 

       @Override

       public void setApplicationContext(ApplicationContext context) throws BeansException {

              this.context = context;

             

       }

 

       @Override

       public void setBeanName(String beanName) {

              System.out.println("Bean Name is "+beanName);

             

       }

      

}
