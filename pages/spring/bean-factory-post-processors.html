Bean Factory Post Processors

One of the extensions of spring is bean factory post processors. We can write our own bean factory post processor. Bean factory provides the entire bean. To implement bean factory post processor need to write a separate class.

MyBeanFactoryPostProcessor.java

 

public class MyBeanFacoryPostProcessor implements BeanFactoryPostProcessor{

 

       @Override

       public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFacotry) throws BeansException {

              System.out.println("BeanFacoryPost Processor Executing ");

       }

}

bean.xml

<?xml version="1.0" encoding="UTF-8"?>

<beans xmlns="http://www.springframework.org/schema/beans"

       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

 

<bean id="triangle" class="com.anu.tqa.beanfactorypostprocessor.Triangle">

       <property name="pointA" ref="pint0"/>

       <property name="pointB" ref="pint1"/>

       <property name="pointC" ref="pint2"/>

</bean>

 

<bean id="pint0" class="com.anu.tqa.beanfactorypostprocessor.Point">

       <property name="pointX" value="0"/>

       <property name="pointY" value="0"/>

</bean>

 

<bean id="pint1" class="com.anu.tqa.beanfactorypostprocessor.Point">

       <property name="pointX" value="0"/>

       <property name="pointY" value="20"/>

</bean>

 

<bean id="pint2" class="com.anu.tqa.beanfactorypostprocessor.Point">

       <property name="pointX" value="20"/>

       <property name="pointY" value="0"/>

</bean>

 

<!-- registering bean factory post processor class here -->

<bean class="com.anu.tqa.beanfactorypostprocessor.MyBeanFacoryPostProcessor"/>

</beans>

 

 

We can override the default functionality of bean factory post processor. One of the example of BeanFactoryPostProcessor is PropertyPlaceHolderConfigurer . Like in beans.xml file different values are abliable. No I want to read those values from a property file.

 PropertyPlaceHolderConfigurer

One of the examples of BeanFactoryPostProcess is PropertyPlaceHolderConfigurer. Now we are going to put some placeholder in bean.xml file instead of value. And we will put those values in a property file. And PropertyPlaceHolderConfigurer will read those values from property file and put those values in the placeholder which is declared in bean.xml file.

pointconfig.properties

pointA.pointX=0

pointA.pointY=0

** Please the pointconfig.properties file in src folder.

 

bean.xml

<?xml version="1.0" encoding="UTF-8"?>

<beans xmlns="http://www.springframework.org/schema/beans"

       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

 

<bean id="triangle" class="com.anu.tqa.beanfactorypostprocessor.Triangle">

       <property name="pointA" ref="pint0"/>

       <property name="pointB" ref="pint1"/>

       <property name="pointC" ref="pint2"/>

</bean>

 

<bean id="pint0" class="com.anu.tqa.beanfactorypostprocessor.Point">

       <property name="pointX" value="${pointA.pointX}"/>

       <property name="pointY" value="${pointA.pointY}"/>

</bean>

 

<bean id="pint1" class="com.anu.tqa.beanfactorypostprocessor.Point">

       <property name="pointX" value="0"/>

       <property name="pointY" value="20"/>

</bean>

 

<bean id="pint2" class="com.anu.tqa.beanfactorypostprocessor.Point">

       <property name="pointX" value="20"/>

       <property name="pointY" value="0"/>

</bean>

 

<!-- registering bean factory post processor class here

<bean class="com.anu.tqa.beanfactorypostprocessor.MyBeanFacoryPostProcessor"/> -->

 

<!-- PropertyPlaceHolderConfigurer register here to read the property file -->

<bean class="org.springframework.beans.factory.config.PropertyPlaceholderConfigurer">

       <property name="locations" value="pointconfig.properties"/>

</bean>

</beans>
