  1. Dependency Injection

Dependency injection is some time called as dependency inversion. This is basically decoupled of two objects which is dependent to each other. Dependency Injection happens because there is a container. Spring container contains the bean object. The container instantiate the object, manage the object life cycle of the object and destroy the object.

  1. Spring Container

Spring container is the container of bean. So the pojo that we have created can be containing in spring container. And spring mange all those object in the container. It handles the life cycle of the object, handles the instantiate of the object and destruction of the object. We can have object outside of the spring container, but facility to keep object inside the container is that spring mange the object by its own.

  1. Bean Factory

Spring has a bean factory. We can use bean factory in order to create spring lets create an object for us. So if we do so, bean factory will read the spring.xml, the definition of the class, here bean.xml is the blueprint of the object. So advantage here is that spring creates the object and spring knows about the object.

Bean factory is an interface and there are many class implementations are available like the XMLBeanFactory.

BeanFactory beanFactory = new XMLBeanFactory (new FileSystemResource (“Beans.xml”));

3.1 Main Class

public class MainDrawingApp {

       public static void main (String[] args) {

             

              // Conventional way of creating object

              /*Triangle triangle = new Triangle ();

              triangle.draw ();*/

             

             

              BeanFactory beanFactory = new XmlBeanFactory(new        FileSystemResource("Beans.xml"));

              Triangle triangle = (Triangle) beanFactory.getBean("triangle");

              triangle.draw ();

       }

}

 

3.2 Triangle Class

public class Triangle {

       public void draw () {

              System.out.println ("Triangle Draw");

       }

}

 

3.3 bean.xml

<?xml version="1.0" encoding="UTF-8"?>

<beans xmlns="http://www.springframework.org/schema/beans"

       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

 

<bean id="triangle" class="org.anu.jqa.beanfactory.Triangle"/>

</beans>

 

  1. Application Context

Application context is the big brother of the BeanFacotry. It does everything that BeanFactory does, but also has some additional functionality like event notification, AOP etc.

ApplicationContext is also an interface like BeanFactory and it has many implementation class available.

ApplicationContext applic ationContext = new ClassPathXMLApplicationContext(“beans.xml”);

Move the beans.xml from root of the application to “src” folder. It takes resources from the classpath. Class path means beans.xml should be in src folder.

 

4.1 Main Class

public class MainClass {

       public static void main(String[] args) {

              ApplicationContext context = new ClassPathXmlApplicationContext("bean.xml");

              Triangle triangle = (Triangle) context.getBean("triangle");

              triangle.draw();

       }

}
