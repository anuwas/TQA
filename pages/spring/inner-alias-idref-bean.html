  1. Inner Bean

If any bean is not going to share with any other bean, then that bean need not declared outside of the dependent reference. It can be declared inside of its dependent.

Like in previous example let's say point may be share by different shape, but point1 and point2 is only use for a triangle, so we could defile point 1 and point2 inside the triangle point B and Point C Directly. This kind of bean declaration one bean inside another bean is called inner bean.

8.1 bean.xml

<?xml version="1.0" encoding="UTF-8"?>

<beans xmlns="http://www.springframework.org/schema/beans"

       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

 

<bean id="triangle" class="com.anu.tqa.innerbean.Triangle">

       <property name="pointA" ref="pint0"/>

       <property name="pointB" >

       <!-- id is not required inner bean , id required only outside bean -->

              <bean  class="com.anu.tqa.innerbean.Point">

                     <property name="pointX" value="0"/>

                     <property name="pointY" value="20"/>

              </bean>

       </property>

       <property name="pointC" >

       <!-- id is not required inner bean , id required only outside bean -->

              <bean  class="com.anu.tqa.innerbean.Point">

                     <property name="pointX" value="20"/>

                     <property name="pointY" value="0"/>

              </bean>

       </property>

</bean>

 

<bean id="pint0" class="com.anu.tqa.innerbean.Point">

       <property name="pointX" value="0"/>

       <property name="pointY" value="0"/>

</bean>

 

</beans>

 

  1. Aliases Bean

 

Bean aliases are means the different names for the same bean. It's the way where we can give the different name for the same bean.

9.1 MainClass

public class MainClass {

       public static void main(String[] args) {

              ApplicationContext context = new ClassPathXmlApplicationContext("bean.xml");

              Triangle triangle = (Triangle) context.getBean("triangle-alias");

              triangle.draw();

       }

}

  1.  IDREF

“ref” is using for reference . a ref is using to refer the name or id of any bean. To restrict the reference only for id “idref” is using. It is one kind of xml validation in spring. So that the reference only working for id not name.

10.1 bean.xml

<?xml version="1.0" encoding="UTF-8"?>

<beans xmlns="http://www.springframework.org/schema/beans"

       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

 

<bean id="triangle" class="com.anu.tqa.idref.Triangle">

       <property name="pointA" >

              <idref bean="pint0"/>

       </property>

       <property name="pointB" >

       <!-- id is not required inner bean , id required only outside bean -->

              <bean  class="com.anu.tqa.idref.Point">

                     <property name="pointX" value="0"/>

                     <property name="pointY" value="20"/>

              </bean>

       </property>

       <property name="pointC" >

       <!-- id is not required inner bean , id required only outside bean -->

              <bean  class="com.anu.tqa.idref.Point">

                     <property name="pointX" value="20"/>

                     <property name="pointY" value="0"/>

              </bean>

       </property>

</bean>

 

<bean id="pint0" class="com.anu.tqa.idref.Point">

       <property name="pointX" value="0"/>

       <property name="pointY" value="0"/>

</bean>

 

</beans>

11.2 Triangle Class

public class Triangle {

      

       private List<Point> points;

 

       public List<Point> getPoints() {

              return points;

       }

 

       public void setPoints(List<Point> points) {

              this.points = points;

       }

 

       public void draw() {

              for(Point pointObj : points) {

                     System.out.println("Triangle Draw ,"

                                  + " Point "+pointObj.getPointX()+","+pointObj.getPointY());

              }     

       }

}
