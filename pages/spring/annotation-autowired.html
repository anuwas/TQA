  1. @Autowired Annotation

@Required annotation is not doing any dependency wiring, whereas @Autowired annotation actually use to wiring bean dependency. This is similar to the autowire by name or by type which we have used in bean.xml file.

This @Autowired is actually autowiring by type. But then we know that autowiring by type is sometime ambiguous; it depends on the what bean or what type of beans we have declared in bean.xml.

However @Autowired has few other tricks. we can make less ambiguous and we can make less configure, what bean get actually injected.

Here also need a BeanPostProcessor

@Autowired annotation first looks for type and it finds only one type even though the name is different, it autowire it. Now if there multiple bean with the same type then it go for if any name is that match then it autowire by name. It actually doing some intelligence guesses work here.

Now there is an another solution, It it has multiple bean with same type and name, then how do I give clue to spring to autowire.

23.1 Circle.java

public class Circle implements Shape{

 

       private Point center;

      

       public Point getPoint() {

              return center;

       }

 

       @Autowired

       public void setPoint(Point center) {

              this.center = center;

       }

 

       @Override

       public void draw() {

              System.out.println("Drawing Circle ");

              System.out.println("Center of the cirecle is X "+center.getPointX()+", Y "+center.getPointY());

             

       }

}
