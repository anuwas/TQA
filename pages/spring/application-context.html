Application Context

Application context is the big brother of the BeanFacotry. It does everything that BeanFactory does, but also has some additional functionality like event notification, AOP etc.

ApplicationContext is also an interface like BeanFactory and it has many implementation class available.

ApplicationContext applic ationContext = new ClassPathXMLApplicationContext(“beans.xml”);

Move the beans.xml from root of the application to “src” folder. It takes resources from the classpath. Class path means beans.xml should be in src folder.

 

 Main Class

public class MainClass {

       public static void main(String[] args) {

              ApplicationContext context = new ClassPathXmlApplicationContext("bean.xml");

              Triangle triangle = (Triangle) context.getBean("triangle");

              triangle.draw();

       }

}

Triangle Class

public class Triangle {

       public void draw() {

              System.out.println("Triangle Draw");

       }

}

Bean.xml

<?xml version="1.0" encoding="UTF-8"?>

<beans xmlns="http://www.springframework.org/schema/beans"

       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

 

<bean id="triangle" class="com.anu.tqa.applicationcontext.Triangle"/>

</beans>
