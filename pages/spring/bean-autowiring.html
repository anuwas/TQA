  1.  Bean Autowiring

Autowiring is the feature provided by the spring framework that helps us to skip some of the configurations that we have to do. We have seen that every member variable have to be configure that are reference to another bean we have to specify the reference explicitly . Spring intelligently guessing the references. Autowiring basically reduce the extra configuration.

When the name of the member variable and the name of the bean property has the same name then auto wiring can be used to set the value for the member variable.

There are different ways are available for autowiring . One of the ways is “byName”.

 

12.1 Main Class

public class MainClass {

       public static void main(String[] args) {

              ApplicationContext context = new ClassPathXmlApplicationContext("bean.xml");

              Triangle triangle = (Triangle) context.getBean("triangle");

              triangle.draw();

       }

}

public class Triangle {

      

       private Point pointA;

       private Point pointB;

       private Point pointC;

      

       public Point getPointA() {

              return pointA;

       }

 

       public void setPointA(Point pointA) {

              this.pointA = pointA;

       }

 

       public Point getPointB() {

              return pointB;

       }

      

       public void setPointB(Point pointB) {

              this.pointB = pointB;

       }

 

       public Point getPointC() {

              return pointC;

       }

      

       public void setPointC(Point pointC) {

              this.pointC = pointC;

       }

 

       public void draw() {

              System.out.println("Triangle Draw ,"

                           + " Point A "+getPointA().getPointX()+","+getPointA().getPointY()

                           + ", Pint B "+getPointB().getPointX()+", "+getPointB().getPointY()

                           + ", Point C "+getPointC().getPointX()+", "+ getPointC().getPointY());

       }

      

}

12.4.1 Autowire by type

<bean id="triangle" class="com.anu.tqa.autowiring.Triangle" autowire="byType">

      

</bean>

 

Another way of autowiring is “byTpe” , and it works only when there is only one member variable and only one bean of each type. It checks the class with the member variable class and ignore the name.

For this particular case autowiring will not work, because there are three point variable. So spring cannot assume what would be the type, as we know that we should have only one member variable required for autowiring by type.

 

12.4.2 Autowire by Constructor

<bean id="triangle" class="com.anu.tqa.autowiring.Triangle" autowire="byConstrucotr">

      

</bean>

 

Autowire by constructor is almost same as autowire by type, only one difference is that for constructor autowiring we need constructor instead of setter method injection.
